#### 进程调度

```shell
# 概念
- 功能:保存现场,挑选进程,恢复现场
- 两级调度(作业调度/进程调度)
- 三级调度(中级调度)
# 调度策略
- 考虑因素
- 评价标准
1.CPU利用率
2.吞吐量
3.周转时间(进入内存/就绪等待/CPU完成IO)
	带权周转时间(周转时间/实际运行时间)
4.就绪等待时间
5.响应时间
```

```shell
# 经典进程调度算法
# 先来先服务FCFS
- 作业调度:
	1.选择队头一或几个作业(后备作业队列)
	2.调入内存,分配相应资源
	3.创建进程,放入就绪队列(队尾)
- 进程调度:
	1.选择一个最先进入该队进程(就绪队列/队头)
	2.把CPU分给它,令其投入运行.
- 有利于长作业
- 有利于CPU繁忙型作业(IO>就绪)
# 时间片轮转调度RR(分时操作系统)
- 进程调度:
	1.分配队首时间片
	2.时间片用完(时钟中断)>队尾
	3.再分配队首时间片
- 时间片长短:
	-系统响应时间
	-就绪队列进程数目
	-进程转换时间
	-CPU运行指令速度
# 优先级调度(系统内部定义/外部指定/优先数)
- 抢占式/非抢占式优先级法
- 静态优先级(开销小/饥饿)
- 动态优先级(运行>降低/等待>渐升)
# 最短作业优先调度SJF(非抢占式)
- 预测执行长度
- 降低平均等待时间(对长作业不利/紧迫性作业)
# 最短剩余时间优先调度SRTF(抢占式)
- 新短作业(系统开销)
# 多级队列调度
- 作业特性(占用内存/作业类型)>不同队列(各自算法)
- 前台优先级>后台
# 多级反馈队列调度
1. 队列优先级依次降低
2. 时间片依次加倍
3. 时间片未完>下一个队列(等待多个时间片)>回到高优先级

# 公平共享调度(用户组/权值)
- 监视使用情况
# 保证调度
- n用户获得CPU1/n
# 彩票调度
- 概率算法
- 防止饥饿
```
