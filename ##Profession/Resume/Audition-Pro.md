## FrontEnd

#### 数字化巡店与质量追溯系统（移动端/小程序）

背景： 连锁餐饮最怕“食品安全”和“门店标准走样”。总部需要一个工具监控全国门店。
技术点： 离线拍照、断点续传、水印相机、海量图片加载优化。
●项目描述： 针对连锁品牌总部开发的数字化巡店系统，涵盖卫生检查、食材溯源、员工考勤等功能。
●核心攻关：
○高频多图上传： 针对后厨信号差的特点，开发了分片上传与断点续传机制，并实现了图片在前端自动压缩（Quality/Size 均衡），确保 10MB 的巡店照片在弱网下也能快速上报。
○离线表单处理： 采用 PWA (Progressive Web App) 理念，支持督导在冷库、地下室等无网环境下完成巡店录入，并在网络恢复后利用后台同步（Background Sync）自动提交。
○动态动态表单： 针对不同业态（火锅、快餐、咖啡）的检查标准差异，设计了一套由 JSON 驱动的高度动态化表单配置引擎。

#### 多端营销看板与数据大屏（数据可视化）

背景： 老板需要随时随地看到营收、翻台率、客单价。
技术点： 响应式布局、Canvas/SVG、复杂报表导出、数据安全。
●项目描述： 为品牌创始人及区域经理打造的决策支持系统（决策驾驶舱），支持 App、H5 以及门店大屏多端展示。
●核心攻关：
○响应式可视化方案： 利用一套代码适配门店 55 寸监控大屏和管理者手机端，通过 CSS Transform 缩放方案 解决了不同分辨率下的 ECharts 图表失真问题。
○极端报表优化： 处理跨度为 1 年、涉及百万级流水的导出需求，前端通过 Web Streams API 实现流式下载，避免浏览器因内存溢出而崩溃。
○多维度对比： 实现了“同比、环比、区域比”的复杂交互逻辑，支持用户在地图上通过点击不同商圈实时查看营收热力分布。

#### 扫码点餐“秒开”与性能调优（C端/流量型）

背景： 点餐高峰期，扫码后的白屏时间直接影响下单率。
技术点： 骨架屏、预加载、SSR/SSG、按需加载、缓存策略。
●项目描述： 对公司核心 C 端扫码点餐小程序进行性能重构，目标是解决高峰期页面卡顿与首屏加载慢的问题。
●核心攻关：
○静态资源分发优化： 配合 CDN 预热与 Brotli 压缩，将小程序首包体积控制在 2MB 以内，并对非核心逻辑进行分包异步加载。
○数据预取方案： 基于用户扫码的桌号 ID，在页面初始化前通过 Pre-fetch 提前获取门店菜单数据，将点餐页从“白屏”到“可交互”的时间 (TTI) 降低了 60%。
○长列表渲染： 针对菜品超过 500 个的超大型菜单，利用虚拟列表机制确保滑动顺畅度保持在 60fps。

#### 项目一：全渠道餐饮收银系统 (POS)

●	技术栈： Electron, React, MobX, SQLite, WebSocket
●	挑战： 餐饮高峰期系统压力大，且需同时连接打印机、副屏、扫码枪等多种外设。
●	解决方案：
○	采用 多进程架构，将业务逻辑与硬件通讯进程分离，避免打印卡顿导致界面死锁。
○	优化 WebSocket 实时接单 机制，引入心跳重连及数据补发逻辑，确保外卖订单零丢失。

#### 项目二：多品牌连锁餐饮管理后台 (SaaS)

●	技术栈： Vue 3, Vite, Ant Design, ECharts
●	挑战： 报表数据量大（千万级流水），权限校验复杂（多级门店管理）。
●	解决方案：
○	实现了基于 RBAC 模型 的细粒度权限控制方案（菜单、按钮、数据行级权限）。
○	针对财务报表，利用 Web Worker 在后台处理复杂运算，避免主线程阻塞，极大提升了用户在大数据量筛选时的体验。

#### 项目三：餐饮后厨调度系统 (KDS - Kitchen Display System)

技术栈： Electron, Vue 3, Pinia, WebSocket, MQTT, Node.js
●	项目描述： 针对连锁餐厅后厨设计的数字化调度终端，实现订单实时显示、智能催菜、分单打印及出菜管理。
●	核心攻关：
○	极致实时性： 采用 MQTT + WebSocket 双重保障机制，确保在高峰期每秒上百单下发时，端到端延迟低于 100ms，并设计了离线消息补发机制，防止因后厨网络波动导致的“丢单”现象。
○	多屏同步与响应： 针对后厨复杂的显示环境，实现了“一主多从”的跨屏联动架构。主厨端操作状态实时同步至配菜端，并利用 Web Workers 处理复杂的订单排序算法（如：按菜品类别聚合、按等候时长预警）。
○	硬件深度集成： 封装了 Electron 打印插件，实现静默分单打印（按档口自动切单），支持各类热敏打印机的 ESC/POS 指令集，并解决了后厨高温环境下打印机频繁断连的自动重连策略。
●	成果： 试点门店出菜效率提升 25%，因错漏单导致的退餐率降低了 15%。

#### 项目四：餐饮供应链 ERP 管理系统 (SCM)

技术栈： React, TypeScript, Ant Design Pro, ECharts, Canvas
●	项目描述： 覆盖从供应商管理、采购申请、入库验收、库存盘点到成本核算的全链路供应链系统。
●	核心攻关：
○	复杂逻辑表单： 针对采购申请中上百个 SKU 的录入需求，开发了高性能可编辑表格，支持 Excel 批量粘贴、实时库存校验与公式计算。通过 虚拟列表 (Virtual List) 解决了万级数据在前端进行复杂联动计算时的性能瓶颈。
○	数据可视化分析： 基于 ECharts 打造了库存预警仪表盘，通过对历史消耗数据的可视化分析，实现了“智能补货建议”的前端呈现，通过动态波形图直观展示损益趋势。
○	流程引擎自定义： 封装了一套轻量级的可视化工作流配置器，允许总部运营人员通过拖拽配置不同品牌、不同地区的采购审批流程，极大地提高了系统的通用性。
●	成果： 实现了全国 500+ 门店的物料数字化流转，库存周转率提升 18%，财务对账周期从 3 天缩短至 0.5 天。

## BackEnd

#### 项目一：餐饮实时接单与排队调度系统 (全栈)

●    **技术栈：** Node.js, NestJS, Redis, Socket.io, React, MySQL

●    **痛点：** 餐厅高峰期并发量大，数据库连接池容易打满，导致前端点餐卡顿。

●    **方案：** * 引入 **Redis 缓存预热机制**，将门店菜单和活动配置常驻缓存，减少 70% 的数据库查询。

○    采用 **WebSocket 集群方案**，结合 Redis 的 Pub/Sub 模式实现跨服务器的消息推送，确保多台收银机、后厨屏状态秒级同步。

○    前端采用 **React 18 并发模式** 优化长列表渲染，确保在高频推单时界面不掉帧。

#### 项目二：多租户餐饮供应链管理平台 (SCM)

●    **技术栈：** Node.js, Prisma ORM, PostgreSQL, Vue 3, Docker

●    **挑战：** 不同餐饮品牌的数据隔离需求极其严格，且核算逻辑复杂。

●    **方案：** * 设计了基于 **Row-Level Security (RLS)** 的数据库多租户方案，从底层物理隔离数据安全。

○    编写了大量的 **Database Migrations** 与种子脚本，通过 Node.js 脚本自动化处理每月千万级的财务结算数据汇总。

○    利用 **Node.js Stream** 技术实现了超大 Excel 报表（100万行+）的流式导出，解决了服务器内存溢出（OOM）问题。

#### 项目三：后厨 KDS 硬件通讯中间件

●    **技术栈：** Electron, Node.js (SerialPort), MQTT, SQLite

●    **方案：** * 开发了一个跨平台的 Node.js 桌面中间件，用于连接后厨的各种硬件（蓝牙秤、标签机）。

○    利用 Node.js 的 **WorkerThreads** 处理硬件心跳检测与协议解析，确保不阻塞UI 渲染主线程。

#### 项目一：全链路支付与账务清算系统（后端/全栈）

**背景：** 餐饮行业对钱极其敏感。聚合支付（微信、支付宝、会员卡、团购券）的对账和清算是 SaaS 系统的核心。

**体现能力：** 事务一致性、分布式锁、安全性、补偿机制。

●    **项目描述：** 负责餐饮 SaaS 平台的支付中台开发，支撑每日数百万笔流水交易。

●    **技术栈：** Node.js (NestJS), MySQL, Redis, RocketMQ, Crypto

●    **核心攻关：**

○    **分布式事务控制：** 针对“扫码-扣款-下单”链路，利用 **TCC（Try-Confirm-Cancel）模式** 确保在网络波动下的数据最终一致性，杜绝“用户扣款但未下单”的情况。

○    **高并发对账：** 面对海量交易流水，利用 Node.js **Worker Threads** 进行多线程并行对账，将 T+1 对账耗时从小时级缩短至分钟级。

○    幂等性设计： 设计了基于 Redis + Token 的全局幂等机制，有效防止因用户重复点击或网络重试导致的重复支付。

#### 项目二：餐饮私域流量营销引擎（全栈/大数据）

**背景：** 餐饮不仅是线下点餐，更需要线上的优惠券推送、会员等级、积分商城等。

**体现能力：** 复杂业务建模、规则引擎、高性能查询。

●    **项目描述：** 从 0 到 1 构建的一套支撑千万级会员的营销中台。

●    **技术栈：** Koa, MongoDB (存储非结构化规则), Redis, React

●    **核心攻关：**

○    **动态规则引擎：** 针对“满 X 元减 Y 元”、“会员专享券”、“限时折扣”等复杂叠加逻辑，设计了一套基于**JSON Schema** 的插件化规则引擎，使新活动上线无需改动核心逻辑。

○    **精准触达系统：** 利用 Node.js 的异步非阻塞特性，结合 **RabbitMQ** 实现千万级用户的优惠券秒级发放与 Push 推送。

○    **性能调优：** 针对 B 端商家极其复杂的“营销效果报表”，通过 MongoDB 的 **AggregationPipeline** 优化聚合查询，提升复杂统计的响应速度。

#### 项目三：基于 Node.js 的门店边缘服务器（全栈/架构）

**背景：** 大中型餐厅担心云端宕机导致全场无法收银。在店里部署一台“边缘服务器”做本地热备是高级方案。

**体现能力：** 架构设计、数据同步、跨端部署。

●    **项目描述：** 为解决大型商超弱网环境下收银挂掉的问题，开发的“云边协同”本地服务器方案。

●    **技术栈：** Node.js, SQLite, Docker, MQTT, React

●    **核心攻关：**

○    **双机热备与自动切换：** 当检测到公网断开时，前端 POS 自动切换至本地 Node.js 边缘服务器，利用 **SQLite** 进行本地存储，网络恢复后通过**双向同步算法**处理冲突并回传至云端。

○    **轻量化部署：** 将 Node.js 环境与数据库打包进 **Docker** 镜像，实现了一键式在各类工业 PC 或低功耗硬件上部署，极大降低了门店维护成本。

○    **多端同步：** 利用 **MQTT** 协议实现店内 POS、厨显、打印机的实时消息总线，确保本地网络下的极致响应。
